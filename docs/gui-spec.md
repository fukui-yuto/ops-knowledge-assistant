# Web GUI 仕様書

## 1. 概要

ユーザーがコマンドやコードを一切使わずに、ブラウザだけで以下の操作を完結できるWeb GUIを提供する。

- ナレッジ（過去手順・チケット等）の追加・閲覧・削除
- テンプレートの閲覧・管理
- 新規手順書の生成
- ナレッジの検索

## 2. 技術選定

| 項目 | 技術 | 理由 |
|---|---|---|
| フレームワーク | Streamlit | Pythonのみ、追加フロントエンド不要、高速プロトタイピング |
| エントリポイント | app.py | `streamlit run app.py` で起動 |
| ポート | 8502 | プロジェクトデフォルト |

## 3. 画面構成

サイドバーでページを切り替えるシンプルな構成。
サイドバーにはナレッジ自動同期（watchdog）のステータスと最終同期結果も表示される。

```
┌──────────────┬──────────────────────────────────┐
│  サイドバー    │          メインエリア              │
│              │                                  │
│  📄 手順書生成 │  （選択中のページが表示される）      │
│  📚 ナレッジ管理│                                  │
│  📋 テンプレート│                                  │
│  🔍 検索      │                                  │
│              │                                  │
│  ──────────  │                                  │
│  ⚙ 設定      │                                  │
└──────────────┴──────────────────────────────────┘
```

## 4. 画面別仕様

### 4.1 手順書生成ページ（メイン画面）

**目的**: タイトルを入力するだけで手順書を生成する

**レイアウト**:
```
┌──────────────────────────────────────────┐
│ 手順書を生成する                           │
│                                          │
│ タイトル: [________________________]  ← 必須 │
│                                          │
│ ▶ 詳細オプション（折りたたみ）              │
│   説明:     [________________________]    │
│   テンプレート: [default     ▼]           │
│   追加情報:  [________________________]    │
│   参照数:   [3]                           │
│                                          │
│         [ 生成する ]                      │
│                                          │
│ ─── 生成結果 ──────────────────────────── │
│                                          │
│ # PostgreSQL バックアップ手順                 │
│ ## 概要                                   │
│ ...（Markdownプレビュー）                  │
│                                          │
│ [ コピー ] [ ダウンロード ] [ ナレッジに追加 ] │
└──────────────────────────────────────────┘
```

**操作フロー**:
1. タイトルを入力して「生成する」をクリック
2. 生成中はスピナーを表示
3. 結果がMarkdownプレビューで表示される
4. 「コピー」でクリップボードにコピー
5. 「ダウンロード」で .md ファイルとしてダウンロード
6. 「ナレッジに追加」で生成結果をそのままナレッジベースに取り込む

**詳細オプション**（折りたたみ、通常は非表示）:
- 説明: 省略可。補足説明を入力
- テンプレート: ドロップダウンで選択（デフォルト: 自動選定）
- 追加情報: 対象サーバー名や制約条件など
- 参照数: 参考にする過去手順の数（1〜10）

### 4.2 ナレッジ管理ページ

**目的**: 過去手順・チケットの追加・閲覧・削除

**レイアウト**:
```
┌──────────────────────────────────────────┐
│ ナレッジ管理                              │
│                                          │
│ ── ファイルアップロード ───────────────── │
│                                          │
│ 種別:     [wiki ▼]                       │
│                                          │
│ [ ファイルをドラッグ＆ドロップ ]             │
│ または [ ファイルを選択 ] (.md)             │
│                                          │
│         [ アップロード ]                   │
│                                          │
│ ── 登録済みナレッジ一覧 ────────────────── │
│                                          │
│ フィルタ: 種別 [全て ▼]                   │
│                                          │
│ ┌─────┬────────────┬──────┬────┐         │
│ │ 種別 │ タイトル     │chunks│ 操作 │         │
│ ├─────┼────────────┼──────┼────┤         │
│ │wiki │サーバー構築  │  5   │ 🗑  │         │
│ │wiki │バックアップ  │  8   │ 🗑  │         │
│ │issue│Pod再起動    │  1   │ 🗑  │         │
│ └─────┴────────────┴──────┴────┘         │
│                                          │
│ 合計: 3件 / 14 chunks                     │
└──────────────────────────────────────────┘
```

**操作フロー**:

アップロード:
1. 種別（source_type: wiki / issue）をドロップダウンで選択
2. Markdownファイルをドラッグ＆ドロップまたは選択
4. 「アップロード」をクリック
5. タイトルはファイル内の `# 見出し` から自動抽出、external_id はファイル名から自動生成
6. 結果（追加/更新/スキップ）を表示

削除:
1. 一覧の 🗑 ボタンをクリック
2. 確認ダイアログ「本当に削除しますか？（DB・ChromaDB・ファイルから完全に削除されます）」
3. 「削除する」で PostgreSQL / ChromaDB / LocalStorage から連動削除

閲覧:
1. タイトルをクリックすると詳細パネルが開く
2. 全文とメタデータを表示

### 4.3 テンプレートページ

**目的**: テンプレートの一覧表示・内容確認

**レイアウト**:
```
┌──────────────────────────────────────────┐
│ テンプレート管理                           │
│                                          │
│ ── テンプレート一覧 ────────────────────── │
│                                          │
│ ┌──────────┬──────────────────────────┐  │
│ │ 名前      │ 説明                     │  │
│ ├──────────┼──────────────────────────┤  │
│ │ default  │ 汎用手順書テンプレート     │  │
│ │ k8s      │ Kubernetes操作手順       │  │
│ │ network  │ ネットワーク作業手順      │  │
│ └──────────┴──────────────────────────┘  │
│                                          │
│ ── プレビュー: default.md ──────────────  │
│                                          │
│ # {{title}}                              │
│ ## 概要                                   │
│ - 対象システム: {{system}}                 │
│ ...（Markdownプレビュー）                  │
│                                          │
│ [ ダウンロード ]                           │
└──────────────────────────────────────────┘
```

**操作フロー**:
1. テンプレート名をクリックするとプレビューが表示される
2. 「ダウンロード」でテンプレートファイルをダウンロードできる
3. テンプレートの追加はGUIからアップロード（ファイル or 直接入力）で可能
4. 既存テンプレートの上書きはチェックボックスで確認後に実行
5. default 以外のテンプレートはGUIから削除可能
6. テンプレートファイルは `data/templates/` に保存され、Docker Compose ではホストと双方向同期される

## 5. 画面遷移

```
起動 → 手順書生成ページ（デフォルト）
       │
       ├── サイドバー → ナレッジ管理ページ
       ├── サイドバー → テンプレートページ
       ├── サイドバー → 検索ページ
       └── サイドバー → 設定ページ

手順書生成 → 「ナレッジに追加」→ ナレッジ管理ページに遷移（追加後）
```

## 6. レスポンス要件

| 操作 | 目標時間 | UX |
|---|---|---|
| ページ遷移 | 即座 | - |
| ナレッジアップロード | 10秒以内 | プログレスバー表示 |
| 手順書生成 | 30秒以内 | スピナー + 「生成中...」メッセージ |
| 検索 | 2秒以内 | スピナー表示 |
| ナレッジ削除 | 3秒以内 | 確認ダイアログ後、成功メッセージ |

## 7. エラー表示

| 状況 | 表示 |
|---|---|
| DB未接続 | 赤帯バナー「PostgreSQLに接続できません。設定を確認してください」 |
| APIキー未設定 | 赤帯バナー「APIキー（GOOGLE_API_KEY または OPENAI_API_KEY）が設定されていません」 |
| 生成失敗 | エラーメッセージ + リトライボタン |
| アップロード失敗 | エラーメッセージ（原因表示） |
| テンプレート不在 | 警告「テンプレートが見つかりません。default.md を使用します」 |

## 8. 起動方法

```bash
# GUI起動
streamlit run app.py

# ポート指定
streamlit run app.py --server.port 8080

# 外部アクセス許可
streamlit run app.py --server.address 0.0.0.0
```

## 10. ユーザー操作フロー（GUIのみ）

### 初回セットアップ
1. `streamlit run app.py` で起動（これだけ）
2. 設定ページで「スキーマ初期化」をクリック
3. ナレッジ管理ページでファイルをアップロード

### 日常操作: ナレッジ追加
1. ナレッジ管理ページを開く
2. 種別（wiki / issue）を選択
3. ファイルをドラッグ＆ドロップ
4. 「アップロード」をクリック

### 日常操作: 手順書生成
1. 手順書生成ページを開く
2. タイトルを入力
3. 「生成する」をクリック
4. 結果を確認し、コピーまたはダウンロード

### 日常操作: ナレッジ削除
1. ナレッジ管理ページの一覧で 🗑 をクリック
2. 確認ダイアログで「削除する」をクリック
