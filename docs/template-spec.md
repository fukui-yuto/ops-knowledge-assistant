# テンプレート仕様書

## 1. 概要

テンプレートは手順書の「構成と記載ルール」を定義するMarkdownファイルである。
LLMはテンプレートの構成に従って新規手順書を生成する。

## 2. 配置ルール

| 項目 | 内容 |
|---|---|
| 配置先 | `data/templates/` |
| ファイル形式 | Markdown (.md) |
| 命名規則 | `{カテゴリ名}.md`（例: `default.md`, `network.md`, `k8s.md`） |
| エンコーディング | UTF-8 |
| バージョン管理 | Git管理（data/templates/ はgitignore対象外） |

## 3. テンプレート構成

### 3.1 必須セクション

全テンプレートに以下のセクションを含めること:

| セクション | 見出しレベル | 目的 |
|---|---|---|
| タイトル | `#` | 手順書のタイトル |
| 概要 | `##` | 対象システム・目的・所要時間・作業者レベル |
| 前提条件 | `##` | 作業開始前に満たすべき条件 |
| 事前確認 | `##` | 作業前のチェック項目 |
| 作業手順 | `##` | メインの手順（Step N: で番号付け） |
| 作業後確認 | `##` | 作業完了後のチェック項目 |
| 切り戻し手順 | `##` | 問題発生時のロールバック手順 |
| 備考 | `##` | 注意事項・補足情報 |

### 3.2 オプショナルセクション

| セクション | 用途 |
|---|---|
| 関連ドキュメント | 参照先リンク |
| 変更履歴 | 改訂記録 |
| 承認欄 | レビュー・承認情報 |
| 影響範囲 | 作業による影響を受けるシステム一覧 |

## 4. プレースホルダ

テンプレート内で以下のプレースホルダを使用できる。LLMが生成時に適切な値で置換する。

| プレースホルダ | 説明 | 例 |
|---|---|---|
| `{{title}}` | 手順書タイトル | Proxmox バックアップ手順 |
| `{{system}}` | 対象システム名 | Proxmox VE |
| `{{purpose}}` | 作業目的 | 全VMの日次バックアップ設定 |
| `{{duration}}` | 想定所要時間 | 30分 |
| `{{level}}` | 作業者レベル | 中級（Proxmox基本操作ができる） |
| `{{date}}` | 作成日 | 2026-04-28 |
| `{{author}}` | 作成者 | ops-knowledge-assistant |

## 5. テンプレートカテゴリ

用途別にテンプレートを用意する:

| テンプレート名 | 用途 | 特徴 |
|---|---|---|
| `default` | 汎用手順書 | 基本的なセクション構成 |
| `network` | ネットワーク作業 | NW構成図セクション、疎通確認手順を含む |
| `k8s` | Kubernetes操作 | kubectl コマンド例、Pod/Service確認を含む |
| `storage` | ストレージ作業 | 容量確認、バックアップ確認を含む |
| `security` | セキュリティ対応 | 影響調査、証跡保全、報告セクションを含む |
| `monitoring` | 監視設定 | アラート定義、閾値設定、テスト手順を含む |
| `migration` | データ移行 | 移行前後の整合性確認、データ検証を含む |
| `incident` | インシデント対応 | タイムライン、影響範囲、根本原因分析を含む |

## 6. テンプレート記述ガイドライン

### コマンド例の書き方
```markdown
### Step 1: 対象サーバーにSSH接続
` ` `bash
ssh user@target-server
` ` `
> 接続できない場合は「切り戻し手順」を参照
```

### チェックリストの書き方
```markdown
## 前提条件
- [ ] 対象サーバーへのSSHアクセス権がある
- [ ] 作業承認が取得済みである
- [ ] バックアップが完了している
```

### 注意書きの書き方
```markdown
> **WARNING**: この操作はサービス停止を伴います。メンテナンスウィンドウ内で実施してください。
```

### 不明箇所の書き方
```markdown
- 対象サーバー: `TODO: 要確認 - 本番環境のホスト名を記入`
```

## 7. テンプレートの追加手順

1. `data/templates/` に新しい `.md` ファイルを作成する
2. 必須セクション（3.1）を含むMarkdownを記述する
3. `python -m scripts.generate_cli --list-templates` で認識されることを確認する
4. Git にコミットする
