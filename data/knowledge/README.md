# knowledge/ — ナレッジ配置ディレクトリ

ここにMarkdownファイルを置くだけでナレッジベースに取り込まれます。
GUI（Streamlit）起動中は watchdog が自動検知して同期します。手動の場合は `sync.py` を実行してください。

## ディレクトリ構成

```
knowledge/
├── procedure/          # 手順書（過去の運用手順書）
│   ├── confluence/     #   出元: Confluence から取得した手順
│   ├── internal/       #   出元: 社内で作成した手順
│   └── {任意}/         #   出元システム名は自由に命名可能
├── ticket/             # チケット（障害対応・インシデント記録）
│   └── jira/           #   出元: JIRA チケット
├── config/             # 構成情報（システム構成・設定ドキュメント）
│   └── k8s/            #   出元: Kubernetes 関連
└── log/                # ログパターン（エラーパターンと対処法）
    └── app/            #   出元: アプリケーション
```

## 種別（第1階層）の説明

| ディレクトリ | source_type | 用途 | 例 |
|---|---|---|---|
| `procedure/` | procedure | 過去の運用手順書 | サーバー構築手順、バックアップ手順、デプロイ手順 |
| `ticket/` | ticket | 障害対応・インシデント記録 | JIRA チケット、障害対応報告書 |
| `config/` | config | システム構成情報 | K8s クラスタ構成、ネットワーク構成図 |
| `log/` | log | ログパターン・エラー対処法 | よくあるエラーと対処法、ログ解析手順 |

上記4種別以外のディレクトリは無視されます。

## 出元システム（第2階層）の説明

自由に命名できます。ナレッジの出元を分類するためのラベルです。

例: `confluence`, `jira`, `internal`, `aws`, `gcp`, `k8s`, `redmine` など

## ファイル形式

- Markdown（`.md`）のみ対応
- ファイルの先頭に `# タイトル` を書くとタイトルとして自動抽出
- タイトルがない場合はファイル名がタイトルになる
- ファイル名（拡張子除く）が `external_id` として使用される

## 自動同期の仕組み

- **GUI起動中**: watchdog がファイルの追加・変更・削除を検知し、3秒後に自動同期
- **手動同期**: `uv run python sync.py`（または `docker compose exec app uv run python sync.py`）
- **差分検知**: ファイルの SHA256 ハッシュで変更を検知。未変更ファイルはスキップ
